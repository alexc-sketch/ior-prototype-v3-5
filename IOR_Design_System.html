Non-Negotiable Design Rules

These 5 rules apply to every file, every page, every component. No exceptions. Violations must be flagged in peer review.

1
No rounded corners — sharp geometry only
border-radius: 0 is set as --radius: 0px in the root variables and applied via the .btn, .ior-pill, and all card/container components. Never override this. The brutalised industrial geometry reflects IOR's heavy infrastructure positioning.
2
No full-bleed hero images with dark overlay
The Homepage is the only exception (per Figma brief — approved). All other pages must use: split layouts (left content, right media), abstract/geometric backgrounds, or reduced-height banners (.ior-hero-banner). The cookie-cutter "dark photo + centred white headline" pattern is explicitly rejected.
3
No Lorem Ipsum — brief copy only
Every text block must use copy provided in the Design Direction document, Copy Scaffolding document, or UX Journey Mapping document. If copy is not provided for a section, insert a .annotation--question bar clearly flagging the gap. Do not fabricate filler copy.
4
Every block must have a full-width annotation bar
The .annotation component (defined in this file) must appear immediately above every content block in every module file. Format: BLOCK [N] — [NAME] · [SPEC] · [DESIGN INTENT]. See the Annotation System section for full usage rules.
5
No generic AI layout patterns — follow the brief block order exactly
The block order for every page is specified in the Copy Scaffolding and Design Direction documents. Follow it exactly. If a page's brief specifies "Interactive Split-Content Block" — implement the .ior-split-interactive component. Do not substitute a simpler static grid. If unsure, flag with .annotation--question.

Import Guide — How to Use This File in Each Module

Copy the entire <style> block from this file into the <head> of each module file. Do not link externally (wireframe self-containment). Then follow this shell structure for every module:

<!-- MODULE FILE SHELL — copy into every .html module --> <!-- Fill in MODULE_NAME and FILE_NAME before saving --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <title>[Page Title] — IOR Wireframe</title> <style> /* === PASTE FULL IOR DESIGN SYSTEM CSS HERE === */ /* Includes); }); /* Accordion */ document.querySelectorAll('.ior-accordion__trigger').forEach(trigger => { trigger.addEventListener('click', () => { trigger.closest('.ior-accordion__item').classList.toggle('ior-accordion__item--open'); }); }); /* Interactive split-content */ document.querySelectorAll('.ior-split-interactive__item').forEach((item, idx) => { item.addEventListener('click', () => { const parent = item.closest('.ior-split-interactive'); parent.querySelectorAll('.ior-split-interactive__item').forEach(i => i.classList.remove('ior-split-interactive__item--active')); parent.querySelectorAll('.ior-split-interactive__panel').forEach(p => p.classList.remove('ior-split-interactive__panel--active')); item.classList.add('ior-split-interactive__item--active'); parent.querySelectorAll('.ior-split-interactive__panel')[idx]?.classList.add('ior-split-interactive__panel--active'); }); }); </script> </body> </html>
